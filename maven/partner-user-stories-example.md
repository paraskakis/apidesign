# Maven Partner API User Stories

## Theme 1: Course & Cohort Intelligence

*Focus: Enabling instructors to track their product inventory, performance, and ratings.*

### 1. View Instructor Courses

**User Story:**
"As an Instructor, I want to retrieve a list of my courses so that I can see what I am currently offering and filter by active or past status."

**Technical Specifications:**

* **Resource:** `GET /instructors/{instructorId}/courses`
* **Parameters:**
* `instructorId` (path, slug string)
* `status` (query, enum: `open`, `inProgress`, `past`)
* `limit` (query, integer, default 20)
* `offset` (query, integer, default 0)



### 2. View Cohort Earnings

**User Story:**
"As an Instructor, I want to explicitly request the financial performance of a specific cohort to analyze its profitability without cluttering the general schedule view."

**Technical Specifications:**

* **Resource:** `GET /courses/{courseId}/cohorts/{cohortId}/earnings`
* **Parameters:**
* `courseId` (path, slug string)
* `cohortId` (path, integer)



### 3. View Course Earnings

**User Story:**
"As an Instructor, I want to view the aggregated earnings for a whole course to gauge its total lifetime value."

**Technical Specifications:**

* **Resource:** `GET /courses/{courseId}/earnings`
* **Parameters:**
* `courseId` (path, slug string)



### 4. Analyze Course Ratings

**User Story:**
"As an Instructor, I want to see the average rating for my courses to improve my teaching methods and materials."

**Technical Specifications:**

* **Resource:** `GET /courses/{courseId}/ratings/summary`
* **Parameters:**
* `courseId` (path, slug string)
* `cohortId` (query, integer, optional)

## Theme 2: Student Enrollment & Management

*Focus: Managing the flow of students into cohorts and waitlists, including automation.*

### 5. View Enrolled Students

**User Story:**
"As an Instructor, I want to list all students currently enrolled in a specific cohort so that I can prepare for the upcoming sessions."

**Technical Specifications:**

* **Resource:** `GET /cohorts/{cohortId}/students`
* **Parameters:**
* `cohortId` (path, integer)
* `limit` (query, integer)
* `offset` (query, integer)



### 6. Manage Waitlist (Upsert)

**User Story:**
"As an Instructor, I want to add a student to a course waitlist—automatically creating their record if they are new—so that I can capture interest without manual data entry."

**Technical Specifications:**

* **Resource:** `POST /courses/{courseId}/waitlist`
* **Parameters:**
* `courseId` (path, slug string)
* **Body:**
* `email` (string, required)
* `name` (string, required)




* **Logic:** If the student does not exist, create them "Just-In-Time" before adding to the waitlist.

### 7. Enroll Student (Upsert)

**User Story:**
"As an Instructor, I want to instantly enroll a student into a specific cohort, even if they don't have an account yet, so I can handle late joiners efficiently."

**Technical Specifications:**

* **Resource:** `POST /cohorts/{cohortId}/students`
* **Parameters:**
* `cohortId` (path, integer)
* **Body:**
* `email` (string, required)
* `name` (string, required)




* **Logic:** If the student does not exist, create them "Just-In-Time" before enrolling.

### 8. Drop Student

**User Story:**
"As an Instructor, I want to remove a student from a specific cohort immediately when they drop out."

**Technical Specifications:**

* **Resource:** `DELETE /cohorts/{cohortId}/students/{studentEmail}`
* **Parameters:**
* `cohortId` (path, integer)
* `studentEmail` (path, string)

## Theme 3: Student Search & Insights

*Focus: Deep diving into specific student history and value.*

### 9. Search Student Enrollment History

**User Story:**
"As an Instructor, I want to view a specific student's full enrollment history to understand their journey across my courses."

**Technical Specifications:**

* **Resource:** `GET /students/{studentEmail}/enrollments`
* **Parameters:**
* `studentEmail` (path, string)
* `limit` (query, integer)
* `offset` (query, integer)



### 10. Search Student Earnings

**User Story:**
"As an Instructor, I want to see the total net earnings generated by a specific student so I can understand the lifetime value of my repeat customers."

**Technical Specifications:**

* **Resource:** `GET /students/{studentEmail}/earnings`
* **Parameters:**
* `studentEmail` (path, string)



### 11. Filter Student Ratings

**User Story:**
"As an Instructor, I want to search through student ratings by score or email so I can identify my promoters and detractors."

**Technical Specifications:**

* **Resource:** `GET /courses/{courseId}/ratings`
* **Parameters:**
* `courseId` (path, slug string)
* `minimumRating` (query, integer)
* `maximumRating` (query, integer)
* `studentEmail` (query, string)
